# DAY1：路由基础复习

## 一、核心理念

**二层通信**解决同一链路下的通信，基于 MAC 地址。
**三层通信**解决不同链路下的通信，基于 IP 地址。

> 二层靠 MAC 表，三层靠路由表。

------

## 二、数据包关键字段

**二层字段：**

- 源 MAC 地址：标识发送方设备
- 目的 MAC 地址：标识接收方设备

**三层字段：**

- 源 IP 地址：标识发送方网络位置
- 目的 IP 地址：标识接收方网络位置

------

## 三、转发方式对比

**路由（三层转发）**
查表依据是路由表。路由表条目包含三个要素：目标网段及掩码、下一跳 IP 地址、出接口。

**交换（二层转发）**
查表依据是 MAC 地址表。根据目的 MAC 地址的查找结果，分为三种转发类型：

| 帧类型     | MAC 地址表状态 | 转发方式                   |
| :--------- | :------------- | :------------------------- |
| 已知单播帧 | 查到目的 MAC   | 从指定端口转发             |
| 未知单播帧 | 查不到目的 MAC | 泛洪（除接收口外的所有口） |
| 广播帧     | 目的 MAC 全 F  | 泛洪（除接收口外的所有口） |

**未知单播帧详解**：当交换机收到一个单播帧，但目的 MAC 地址在 MAC 地址表中查不到时，交换机会把这个帧从除接收端口外的所有端口发送出去（在同一 VLAN 内）。常见原因：交换机刚启动表为空、表项老化（默认 300 秒）、目标设备长时间沉默。

**注意**：未知单播帧和广播帧虽然都泛洪，但触发原因不同——未知单播帧是因为**没查到表项**，广播帧是因为**目的 MAC 本身就是广播地址**。

------

## 四、静态路由

### 标准配置格式

text

```
ip route-static 目标网段 掩码 下一跳IP
```



例如：`ip route-static 192.168.1.0 24 10.0.12.2`

### 常见错误：只指定出接口

错误示例：`ip route-static 2.2.2.2 32 g0/0/0`

**问题原因**：只指定出接口时，路由器认为目标 IP 就在本链路，不断发送 ARP 请求询问该 IP 的 MAC 地址。无人回应 → 无法封装 MAC → 链路不通。

**最佳实践**：始终同时指定下一跳 IP 地址。

------

## 五、ARP 代理

### 什么是 ARP 代理

当主机没有配置网关，却需要与同一网段但不同物理网络的主机通信时，ARP 代理让路由器代替目标主机响应 ARP 请求。

**工作过程**：路由器收到 ARP 请求后，确认目标 IP 可达，就用自己接口的 MAC 地址回应。后续数据包都发给路由器，由路由器转发到真正的目标。

- ## 如何开启 ARP 代理

  **配置位置**：三层接口视图（物理接口或 VLANIF）

  **配置命令**：

  ```
  拓扑为R1 g0/0/0 --- R2 g0/0/0,R1上配置了指定下一跳为出接口的路由
  R2：
  interface g0/0/0
   ip address 10.0.12.1 24
   arp-proxy enable
  ```

  

  ------

  ## 为什么开启 ARP 代理后，只写出接口就能工作

  配置：`ip route-static 10.0.13.0 24 g0/0/0`

  **未开启 ARP 代理**：
  R1 在 g0/0/0 发送 ARP 请求“谁是 10.0.13.2？”，无人回应 → 不通

  **开启 ARP 代理**（注意：是在 **R2 的 g0/0/0** 上开启，不是 R1）：
  R2 收到 R1 的 ARP 请求，发现自己能到达 10.0.13.2，就用自己 g0/0/0 的 MAC 回应 → R1 成功封装数据帧发给 R2 → 通

  > 关键：**对端路由器**开启 ARP 代理，替目标 IP 回应 ARP 请求。

## 六、路由递归

### 什么是路由递归

当路由表中某条路由的下一跳 IP 不是直连网段时，路由器拿着这个下一跳 IP 再次查表，直到找到直连接口。这个过程自动完成，叫路由递归。

### 递归成功的例子

**当前 R1 路由表：**

- 10.0.12.0/24 → 直连，出接口 G0/0
- 10.0.23.0/24 → 下一跳 10.0.12.2

**R1 配置：**
`ip route-static 3.3.3.0 24 10.0.23.3`

**路由器自动递归：**

1. 查 3.3.3.0/24 → 下一跳 10.0.23.3
2. 查 10.0.23.3 → 匹配 10.0.23.0/24，下一跳 10.0.12.2
3. 查 10.0.12.2 → 直连，出接口 G0/0 ✅

**最终结果**：去 3.3.3.0/24 的数据从 G0/0 发出，送到 10.0.12.2。

### 递归的好处

减少静态路由配置量，只需关心逻辑下一跳，中间路径由其他路由指引，不用把完整路径写在一行命令里。

## 七、路由协议分类

**静态路由**：手动配置，适合小型网络。

**动态路由**分为两大类：

内部网关协议（IGP），用于同一个自治系统内部：

- RIP：距离矢量协议，配置简单，跳数限制最大15跳
- OSPF：链路状态协议，收敛快，大型网络首选
- ISIS：链路状态协议，运营商网络常用
- EIGRP： Cisco 私有协议，混合型

外部网关协议（EGP），用于不同自治系统之间：

- BGP：主要用于 ISP 和超大型网络之间的路由交换

------

## 八、一句话总结（考点速记）

- 二层通信：靠 MAC 地址表，在广播域内转发
- 三层通信：靠路由表，通过下一跳和递归实现跨网段通信
- 静态路由：避免只写出接口，务必指定下一跳 IP
- ARP 代理：在接口视图下用 `arp-proxy enable` 开启，让路由器替目标回应 ARP 请求
- 路由递归：拿着下一跳 IP 再查表，一层层找到直连接口
