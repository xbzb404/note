# DAY1：OSPF 核心复习

## 一、OSPF 完整工作流程

> **运行流程：邻居建立 → 邻接关系建立 → 同步LSA → 生成LSDB → SPF算路 → 生成路由表**

- 邻居建立：Hello报文协商参数一致，建立邻居关系，仅负责链路保活
- 邻接建立：广播网络选举DR/BDR，建立可同步拓扑的邻接关系
- 拓扑同步：通过DD/LSR/LSU报文交互，完成全网LSA拓扑数据同步
- 库生成：区域LSA同步完成，所有设备生成完全统一的LSDB拓扑库
- 路径计算：各设备以自身为根，运行SPF算法计算最优转发路径
- 路由生成：最优路径下发全局路由表，作为数据转发的最终依据

------

### 1. 邻居表

> **链路保活，不传输拓扑**

- **生成**：Hello报文参数协商一致
- **内容**：邻居Router-ID、状态（2-Way/Full等）、DR/BDR角色、存活时间
- **查看**：`display ospf peer brief`

------

### 2. LSA 链路状态通告

> **拓扑数据单元，非协议报文，包含在LSU中**

- LSA：拓扑数据条目，非报文
- LSU：OSPF传输报文，承载和泛洪LSA
- LSDB：所有LSA的存储集合
- **交互条件**：仅邻接关系可交互LSA

**六类LSA速查：**

- **第一类 路由器LSA**：所有路由器生成，本区域内传播，描述自身直连链路，构建区域拓扑。（本机接口IP地址，DR的接口IP地址）
- **第二类 网络LSA**：DR生成，本区域内传播，描述广播网段上所有路由器，简化拓扑计算。（Router-ID,掩码mask）
- **第三类 汇总LSA**：ABR生成，跨区域传播，将其他区域路由以网段形式通告过来。
- **第四类 ASBR汇总LSA**：ABR生成，跨区域传播，通告ASBR的位置和开销。
- **第五类 外部LSA**：ASBR生成，全AS泛洪，携带域外引入路由，不进末节区域。
- **第七类 NSSA外部LSA**：NSSA内ASBR生成，本区域传播，由ABR转成第五类传出。
- ~~**第六类 组播LSA（废弃）**：早期MOSPF组播专用，现代网络完全废弃，现网无任何应用。~~

------

### 3. LSDB 链路状态数据库

> **同区域一致，仅存拓扑，不参与选路**

- **生成**：邻接建立后，通过DD/LSR/LSU完成LSA同步校验
- **内容**：本区域所有LSA，完整描述全网链路与网段
- **查看**：`display ospf lsdb`

------

### 4. 全局IP路由表

> **独立算路，路由表各设备不同**

- **生成**：以自身为根，基于LSDB运行SPF算法
- **作用**：数据转发唯一依据
- **查看**：`display ip routing-table protocol ospf`

------

## 二、OSPF 区域核心技术规范

> **基于接口划分，Area 0 是防环核心**

- **划分规则**：基于接口划分，单设备多接口可属不同区域（ABR存在前提）
- **骨干规则**：Area 0唯一骨干区，所有非骨干区必须直连Area 0
- **规模建议**：单区域不超过10台设备

------

## 三、OSPF 进程与 AS 核心规范

### 3.1 协议标准原理

> **进程号仅本地有效，与AS无关**

- OSPF无AS字段，进程号仅本地有效，跨设备无需一致
- 区域是AS内部逻辑划分，与进程号无关

------

### 3.2  常见厂家进程机制

> **多进程默认隔离，通过路由引入互通**

- **默认隔离**：不同进程的LSA、邻居、路由完全独立

- **互通方式**：双向路由引入

  ```
  [Huawei] ospf 1
  [Huawei-ospf-1] import-route ospf 2
  
  [Huawei] ospf 2
  [Huawei-ospf-2] import-route ospf 1
  ```

  

- **注意**：引入后内部路由变为外部路由，优先级降低，需防环路

- **工程用途**：路由实例隔离、业务分层、路由策略标识

------

## 四、ABR 与 ASBR

> **ABR管域间，ASBR管域外，可兼任**

- **ABR**：同时连接Area 0与其他区域，负责区域间路由传递、规整与汇总
- **ASBR**：执行路由重分发，将域外路由引入OSPF，无需依托Area 0
- **共存**：两者独立，单设备可兼任

------

## 五、OSPF 复习总结

- IPV4的OSPF是OSPFv2，OSPFv3才能承载IPV6
- LSU是承载和泛洪LSA的唯一传输报文
- SPF算法负责选路，LSA和LSDB仅提供拓扑数据
- 邻居关系仅保活，邻接关系才同步拓扑
- 同区域LSDB一致，各设备独立算路，路由表互不相同
- 区域基于接口划分，Area 0是防域间环路的唯一骨干中转
- 第一/二类构建域内拓扑，第三/四类传递域间路由，第五/七类引入域外路由
- ABR规整域间路由，ASBR引入外网路由，单设备可兼任
- 华为多进程默认隔离，通过路由引入互通，需注意路由降级风险

------

## 六、OSPF 核心名词速查

- **AS**：同一管理域下运行相同IGP协议的设备集合，内外路由边界
- **邻居关系**：Hello协商后的基础关系，仅保活，不传拓扑
- **邻接关系**：邻居进阶状态，可交互LSA、同步拓扑，协议收敛前提
- **LSA**：最小拓扑数据单元，描述链路与路由信息，非报文
- **LSU**：承载和泛洪LSA的OSPF协议报文
- **LSDB**：同区域一致的全网LSA存储集合，仅存拓扑
- **SPF算法**：以自身为根，基于LSDB计算最短转发路径
- **Area**：接口级逻辑划分单元，缩小拓扑范围
- **Area 0**：骨干区域，域间路由唯一中转节点
- **ABR**：连接骨干与非骨干，传递域间路由
- **ASBR**：对接域外协议，引入外部路由
- **OSPF进程**：华为私有实例机制，仅本地有效，默认隔离
- **OSPFv2**：专用于IPv4网络的OSPF协议版本
- **OSPFv3**：重新设计以原生承载IPv6的OSPF协议版本
