# RSTP 快速生成树协议基础概念（Day 16）

## 一、RSTP 概述

RSTP（Rapid Spanning Tree Protocol）由 IEEE 802.1w 定义，是 STP 的改进版。它在 STP 基础上进行了多项优化，**收敛速度更快**，且**向下兼容 STP**。


## 二、RSTP 端口角色

RSTP 在 STP 三种端口角色（RP、DP、Non-DP）的基础上，将 Non-DP（阻塞端口）进一步细分为两种角色，共四种端口角色：

| 端口角色                       | 说明                                      | 数量规则                | 最终状态           |
| ------------------------------ | ----------------------------------------- | ----------------------- | ------------------ |
| **RP（根端口）**               | 去往根桥的最优入口                        | 每台非根桥有且仅有 1 个 | 转发               |
| **DP（指定端口）**             | 离根桥最近的链路出口                      | 每条链路有且仅有 1 个   | 转发               |
| **Alternate Port（替代端口）** | **RP 的备份**，连接至其他交换机的阻塞端口 | 可有多个                | 阻塞（Discarding） |
| **Backup Port（备份端口）**    | **DP 的备份**，因收到自己 BPDU 而阻塞     | 极少出现                | 阻塞（Discarding） |

> 根桥无 RP，所有端口均为 DP。只有 RP 和 DP 能进入转发状态。

### 替代端口（Alternate Port）

替代端口是**根端口的备份**。当根端口失效时，最优的替代端口可直接成为新的根端口，**30 秒** 内完成切换（15 秒侦听 + 15 秒学习）。

### 备份端口（Backup Port）

备份端口是**指定端口的备份**。当指定端口失效后，备份端口可在 **30 秒** 内成为新的指定端口（15 秒侦听 + 15 秒学习）。

> **补充**：备份端口处于阻塞状态，是因为**收到自己发送的 BPDU** 而阻塞。该场景一般出现在通过 Hub 连接的共享链路（同一台交换机的两个端口都接入同一个 Hub，一个口发出的 BPDU 绕回来被另一个口收到）。实际生产环境**极少见**，因 Hub 已基本淘汰。

**为什么非根桥的 DP 也会发 BPDU？**

非根桥的指定接口会从根接口收到根桥 BPDU 后，**重新生成并发出**自己的 BPDU（根桥 ID 不变，网桥 ID 改成自己），因此"收到自己的 BPDU"是可能发生的。

**各端口角色能否发送 BPDU？**

| 端口角色       | 是否发送 BPDU |
| -------------- | :-----------: |
| RP（根端口）   | ✗（只收不发） |
| DP（指定端口） |       ✓       |
| Alternate Port | ✗（只收不发） |
| Backup Port    | ✗（只收不发） |


## 三、RSTP 端口状态

RSTP 将 STP 的**五种状态精简为三种**，按"是否转发流量"和"是否学习 MAC"来划分：

| RSTP 状态              | 转发用户流量 | 学习 MAC |   对应 STP 状态    |
| ---------------------- | :----------: | :------: | :----------------: |
| **Discarding（丢弃）** |      ✗       |    ✗     | 禁用 + 阻塞 + 侦听 |
| **Learning（学习）**   |      ✗       |    ✓     |        学习        |
| **Forwarding（转发）** |      ✓       |    ✓     |        转发        |

> RSTP 收敛更快的原因之一：减少了状态数量，简化了端口迁移逻辑，不再需要严格经过阻塞→侦听→学习→转发的固定顺序。


## 四、RSTP 对 BPDU 处理的改进

### 4.1 BPDU 发送方式

| 协议     | 拓扑稳定后的 BPDU 发送机制                                   |
| -------- | ------------------------------------------------------------ |
| **STP**  | 只有**根桥**按 Hello Time 主动发送；非根桥收到上游 BPDU 后才**触发**发送自己的 BPDU，完全依赖上游触发 |
| **RSTP** | 拓扑稳定后，**所有设备**均按 Hello Time 间隔**自主发送** BPDU，不再依赖上游触发 |

### 4.2 邻居失效检测

| 协议     | 检测时间                   | 机制                                                         |
| -------- | -------------------------- | ------------------------------------------------------------ |
| **STP**  | **20 秒**（Max Age）       | 等待 BPDU 老化超时后才开始重新计算                           |
| **RSTP** | **6 秒**（Hello Time × 3） | 连续 3 个 Hello Time 未收到 BPDU 即视为邻居失效，立即重新计算 |

### 4.3 次优 BPDU 处理

| 协议     | 处理方式                                                     |
| -------- | ------------------------------------------------------------ |
| **STP**  | 只有**指定端口**会立即处理次优 BPDU；其他端口**忽略**，必须等待 Max Age（20 秒）超时后，缓存老化，才能发送自身更优 BPDU 进行重新收敛 |
| **RSTP** | 当端口收到次优 BPDU 时，**立即丢弃**收到的 BPDU，并**立即回应**自身缓存的更优 BPDU，无需等待任何计时器 |

**一句话总结**：

> RSTP 遇到次优 BPDU 立即丢弃并回应自己的更优 BPDU；STP 只有指定端口会处理，其他端口要等 20 秒老化才能回应。


## 五、RSTP 快速收敛机制

### 5.1 根端口快速切换

根端口失效时，最优的 **Alternate Port（替代端口）** 直接成为根端口并进入 Forwarding 状态。

> 原因：Alternate Port 连接的网段上必然存在一个通往根桥的 DP，无需重新选举。
>
> 切换过程：**Discarding → Forwarding**（跳过 Learning）

### 5.2 指定端口快速切换

指定端口失效时，最优的 **Backup Port（备份端口）** 直接成为指定端口并进入 Forwarding 状态。

> 原因：Backup Port 作为 DP 的备份，提供了另一条从根桥到该网段的备份通路。
>
> 切换过程：**Discarding → Forwarding**（跳过 Learning）

### 5.3 边缘端口（Edge Port）

边缘端口连接终端设备（如 PC、服务器），**不参与 RSTP 计算**，直接从 Discarding 进入 Forwarding，实现 **0 秒收敛**。

> 切换过程：**Discarding → Forwarding**

**边缘端口特性**：

- UP/Down **不会触发**拓扑变更通知
- 若收到 BPDU，**立即丧失**边缘端口特性，变为普通 STP 端口参与计算
- 若开启了 **BPDU 保护（BPDU Guard）**，收到 BPDU 直接 **error-down** 关闭端口

### 5.4 P/A（Proposal/Agreement）机制

P/A 机制是为了在链路 UP 时**快速收敛且防止环路**。

- 从发生变化的设备开始，**逐跳向下**执行 P/A 协商
- 整个过程会**短暂中断**该链路上的数据转发（机制本身无法避免）
- 但收敛速度**极快**，通常在**毫秒级**完成

### 快速收敛总结

> **根口坏 → Alternate 直接转发；指定口坏 → Backup 直接转发；边缘端口 → 0 秒转发。**
>
> RSTP 快速收敛的核心就是**绕过 Learning 状态**，直接从 Discarding 切到 Forwarding。


## 六、RSTP 保护机制

### 6.1 BPDU 保护（BPDU Guard）

| 项目           | 说明                                                      |
| -------------- | --------------------------------------------------------- |
| **一句话概括** | 防止边缘端口意外收到 BPDU，避免非法交换机接入扰乱网络拓扑 |
| **配置位置**   | **全局**（作用于所有边缘端口）                            |
| **触发条件**   | 开启了 BPDU 保护的边缘端口收到任何 BPDU 报文              |
| **处理动作**   | **立即将该端口关闭（error-down）**，彻底隔离              |
| **恢复方式**   | 管理员手动 `undo shutdown` 或配置自动恢复定时器           |

> **极简版**：边缘端口收到 BPDU 就直接"封端口"，防非法接入。

**配置命令**：

```
[Huawei] stp bpdu-protection
```

---

### 6.2 根保护（Root Guard）

| 项目           | 说明                                                         |
| -------------- | ------------------------------------------------------------ |
| **一句话概括** | 防止下游交换机通过发送更优 BPDU 抢占根桥地位                 |
| **配置位置**   | **接口视图**（配在指定端口上）                               |
| **触发条件**   | 开启了根保护的指定端口收到**更优**的 RST BPDU                |
| **处理动作**   | 端口**保持 DP 角色**，但状态变为 **Discarding**（不转发数据），阻止下游设备成为根桥 |
| **恢复方式**   | 不再收到更优 BPDU 后，**自动恢复**为转发状态                 |

> **极简版**：收到更优 BPDU 就"赖着不走"，保持 DP 角色但阻塞转发，防根桥被抢。

**配置命令**：

```
[Huawei-GigabitEthernet0/0/1] stp root-protection
```

---

### 6.3 环路保护（Loop Protection）

| 项目           | 说明                                                         |
| -------------- | ------------------------------------------------------------ |
| **一句话概括** | 防止端口因收不到 BPDU 而误判链路正常，导致形成环路           |
| **配置位置**   | **接口视图**（配在根端口或 Alternate 端口上）                |
| **触发条件**   | 根端口或 Alternate 端口长时间收不到上游 BPDU                 |
| **处理动作**   | 根端口 → **进入 Discarding**，角色变为 DP；Alternate 端口 → **保持 Discarding**，角色变为 DP；同时向网管发送告警 |
| **恢复方式**   | 链路恢复后，重新收到 BPDU，**自动协商**恢复原角色和状态      |

**配置命令**：

```
[Huawei-GigabitEthernet0/0/1] stp loop-protection
```

---

### 6.4 TC 保护（TC-BPDU 保护）

| 项目           | 说明                                                         |
| -------------- | ------------------------------------------------------------ |
| **一句话概括** | 防止 TC 报文风暴导致交换机频繁刷新 MAC 表，影响转发性能      |
| **配置位置**   | **全局**                                                     |
| **处理动作**   | 在指定时间窗口内，只处理限定次数的 TC 报文，超出部分**只计数不刷新转发表** |

**配置命令**：

```
[Huawei] stp tc-protection interval 10      // 统计时间窗口（秒），默认 10
[Huawei] stp tc-protection threshold 15      // 窗口内允许处理的最大 TC 次数，默认 15
```


## 七、RSTP 配置命令

### 7.1 全局基础配置

| 配置项         | 命令                          | 说明                                               |
| -------------- | ----------------------------- | -------------------------------------------------- |
| 切换工作模式   | `stp mode rstp`               | 华为默认 MSTP，需手动切换                          |
| 全局启用 STP   | `stp enable`                  | 默认已启用                                         |
| 配置为根桥     | `stp root primary`            | 优先级强制变为 **0**，之后不能再手动改优先级       |
| 配置为备份根桥 | `stp root secondary`          | 优先级强制变为 **4096**，之后不能再手动改优先级    |
| 手动配置优先级 | `stp priority 32768`          | 步长 **4096**，默认 32768                          |
| 路径开销标准   | `stp pathcost-standard dot1t` | 可选 `dot1d-1998` / `dot1t` / `legacy`，全网需一致 |

### 7.2 接口基础配置

| 配置项         | 命令                    | 说明                   |
| -------------- | ----------------------- | ---------------------- |
| 设置接口开销   | `stp cost 200`          | 数值越小越优           |
| 设置接口优先级 | `stp priority 128`      | 步长 **16**，默认 128  |
| 配置边缘端口   | `stp edged-port enable` | **接终端的口必须配置** |

### 7.3 保护功能配置

| 保护类型  | 配置位置 | 命令                                             | 默认状态 |
| --------- | -------- | ------------------------------------------------ | -------- |
| BPDU 保护 | 全局     | `stp bpdu-protection`                            | 关闭     |
| 根保护    | 接口视图 | `stp root-protection`                            | 关闭     |
| 环路保护  | 接口视图 | `stp loop-protection`                            | 关闭     |
| TC 保护   | 全局     | `stp tc-protection interval 10` / `threshold 15` | 默认开启 |

### 7.4 约束条件速记

| 约束                      | 说明                                                 |
| ------------------------- | ---------------------------------------------------- |
| **根桥锁定**              | `stp root primary` 后，不能再手动 `stp priority ×××` |
| **根保护 + 环路保护互斥** | 同一端口**不能同时**开启根保护和环路保护             |
| **开销标准需一致**        | 全网路径开销计算标准必须一致，否则选路结果可能不一致 |


## 八、典型配置组合

### 最简生产环境

```
[Huawei] stp mode rstp
[Huawei] stp root primary
[Huawei] interface GigabitEthernet 0/0/1
[Huawei-GigabitEthernet0/0/1] stp edged-port enable
```

### 加固防攻击（根保护 + BPDU 保护）

```
[Huawei] stp mode rstp
[Huawei] stp root primary
[Huawei] stp bpdu-protection

[Huawei] interface GigabitEthernet 0/0/24
[Huawei-GigabitEthernet0/0/24] stp root-protection

[Huawei] interface GigabitEthernet 0/0/1
[Huawei-GigabitEthernet0/0/1] stp edged-port enable
```

### 防频繁拓扑震荡（TC 保护调优）

```
[Huawei] stp tc-protection interval 10
[Huawei] stp tc-protection threshold 10
```


## 九、边缘端口 vs 关闭 STP 的区别

| 方式                 | 命令                    | 效果                                                         | 说明                                |
| -------------------- | ----------------------- | ------------------------------------------------------------ | ----------------------------------- |
| **边缘端口（推荐）** | `stp edged-port enable` | 端口 0 秒进入转发，但**仍参与** STP 计算（收到 BPDU 会丧失边缘特性） | **更优**，保留了 STP 的安全检测能力 |
| **关闭 STP**         | `undo stp enable`       | 端口完全不参与 STP，**无任何防环保护**                       | 功能类似但**缺乏安全机制**，不推荐  |

> **建议**：始终使用 `stp edged-port enable`，而非 `undo stp enable`，以保留 STP 的安全防护能力。


## 十、RSTP vs STP 关键差异速查

| 对比项              | STP              | RSTP                  |
| ------------------- | ---------------- | --------------------- |
| 端口状态数量        | 5 种             | 3 种                  |
| 端口角色数量        | 3 种             | 4 种                  |
| BPDU 发送（稳定后） | 仅根桥主动发     | **所有设备**自主发    |
| 邻居失效检测        | 20 秒（Max Age） | **6 秒**（Hello × 3） |
| 次优 BPDU 处理      | 被动等 20 秒老化 | **立即**丢弃并回应    |
| 收敛时间            | 约 30~50 秒      | **毫秒级~数秒**       |
| 边缘端口            | 无               | 支持（0 秒收敛）      |
| P/A 机制            | 无               | 支持（毫秒级收敛）    |
