# MSTP 多生成树协议基础（Day 16）

## 一、MSTP 产生的背景

STP/RSTP 基于**物理接口**生成唯一的生成树，所有 VLAN 共用一棵树，无法实现不同 VLAN 的流量负载分担。

MSTP（IEEE 802.1s）通过引入**实例（Instance）** 概念，将 VLAN 映射到不同的生成树实例，实现基于 VLAN 的流量分流。

> 默认存在实例 0，所有 VLAN 默认归属于实例 0（类似 VLAN 1 的缺省机制）。


## 二、MST 域（MST Region）

MST 域将网络划分为多个独立的计算区域，域内独立计算生成树，域间通过实例 0 互联，防止局部变动引发全网震荡。

**同一个 MST 域的设备必须满足：**

1. 均启用 MSTP
2. **域名（Region Name）** 相同
3. **VLAN 与实例映射关系** 完全相同
4. **修订级别（Revision Level）** 相同

> 以上四项配置不一致的设备会被视为处于不同域。


## 三、核心概念速查

| 缩写     | 全称             | 说明                                                         |
| -------- | ---------------- | ------------------------------------------------------------ |
| **MSTI** | 多生成树实例     | 域内除实例 0 外的其他实例，每个实例关联特定 VLAN，独立计算生成树 |
| **IST**  | 内部生成树       | 实例 0 在域内计算出的生成树                                  |
| **CST**  | 公共生成树       | 连接所有 MST 域的实例 0 生成树，域间仅通过 IST 互联形成 CST  |
| **CIST** | 公共与内部生成树 | IST + CST 的整体，即全网统一的生成树                         |
| **SST**  | 单生成树         | 域内只有一个交换设备时，该设备自身构成 SST                   |

> **实例 0 是 MSTP 的核心枢纽**：域内负责 IST，域间负责 CST，最终形成全网 CIST。


## 四、设备角色

| 角色                          | 说明                                                         |
| ----------------------------- | ------------------------------------------------------------ |
| **总根（CIST Root）**         | 整个 MSTP 网络（所有域）的根桥，全局唯一                     |
| **IST 域根（Regional Root）** | 域内实例 0 中**距离总根最近**的交换设备（按桥 ID 比较），代表本域与总根通信 |
| **MSTI 域根**                 | 域内某多生成树实例（非实例 0）的根桥，负责该实例内部路径选择 |
| **主桥（Master Bridge）**     | 即 IST 域根；若总根本身在域内，则总根即为主桥                |

> IST 域根 ≡ 主桥。


## 五、端口角色

MSTP 端口角色与 RSTP 一致，包括：

- 根端口（RP）
- 指定端口（DP）
- 替代端口（Alt）
- 备份端口（Backup）
- 边缘端口（Edge Port）


## 六、MSTP 报文关键字段

| 字段                        | 长度   | 取值说明                              |
| --------------------------- | ------ | ------------------------------------- |
| Protocol Version Identifier | 1 字节 | STP = 0，RSTP = 2，MSTP = **3**       |
| BPDU Type                   | 1 字节 | MSTP 使用特定 BPDU 类型以携带实例信息 |

> MSTP BPDU 额外携带域配置摘要（M5 摘要），用于校验邻居是否属于同一域。


## 七、配置示例

```
stp region-configuration
 region-name test                // 域名
 revision-level 100               // 修订级别
 instance 1 vlan 2 to 3           // VLAN 2-3 映射至实例 1
 active region-configuration      // 激活配置（必须执行）
```

**根桥 / 备份根桥配置：**

```
// 指定 SW1 为实例 0 和实例 1 的主根，实例 2 的备份根
stp instance 0 root primary
stp instance 1 root primary
stp instance 2 root secondary

// 指定 SW2 为实例 2 的主根，实例 1 的备份根
stp instance 2 root primary
stp instance 1 root secondary
stp instance 0 root primary
```

> 通过为不同实例指定不同的根桥，可实现 VLAN 流量的路径分离与负载分担。


## 八、多域互联工作原理

多域互联场景中：

- 各域内实例独立计算生成树，互不干扰；
- 不同域的相同实例编号（如实例 1）是完全独立的两棵树，各自独立选举根桥、计算阻塞端口；
- 域间仅通过各域的 IST（实例 0）互联，所有域的 IST 共同形成 CST，确保全域无环；
- 域间通信路径由各域的 IST 域根（主桥）负责决定，总根作为全网 CIST 的根桥统筹全局。


## 九、核心优势总结

1. **流量负载分担**：不同 VLAN 映射不同实例，各自优选路径。
2. **故障隔离**：域内变化不影响其他域。
3. **管理灵活**：实例与 VLAN 映射可按需设计，降低运维复杂度。
4. **兼容性好**：MSTP 向下兼容 STP/RSTP。
