# DAY16：STP基础概念补充随笔

为什么需要STP？

在复杂网络中，会在设备之间建立多条物理链路来做**冗余备份**，偶然和必然会导致环路的产生，环路会到网络风暴，也会导致出现



BPDU的比较原则：

`STP`按照如下顺序选择最优的配置`BPDU`：

1. 最小的根桥ID
2. 最小的RPC（链路总开销）
3. 最小的网桥ID
4. 最小的接口ID

## 配置BPDU四个核心参数

**1. 根桥ID**

- 是什么：老大的身份证（优先级 + MAC）
- 作用：确认谁是根桥

**2. RPC（根路径开销）**

- 是什么：从发送端口到根桥的总路费
- 作用：判断哪个口离根桥最近
- 数值参考：100M = 19，10M = 100，越小越优

**3. 网桥ID（发送者桥ID）**

- 是什么：发送BPDU的那台交换机自己的身份证（优先级 + MAC）
- 作用：RPC相同时继续比，小者胜
- 和根桥ID的区别：根桥ID全网不变，网桥ID每台不同

**4. 接口ID（发送端口ID）**

- 是什么：发送BPDU的那个端口的编号（优先级 + 端口号）
- 作用：前三项都相同时最后兜底，小者胜
- 注意：选根接口时比较的是根桥自己的端口ID

**比较顺序**

> 根桥ID → RPC → 网桥ID → 接口ID
>
> （通过接口ID选择根接口时，是比较的根设备的接口ID，因为此时的BPDU是根设备发送的）

对应了上面的BPDU选优规则

当一条链路的速览越大，带宽越小

| 链路带宽     | 对应的STP开销 |
| :----------- | :------------ |
| **10 Mbps**  | **100**       |
| **100 Mbps** | **19**        |
| 1 Gbps       | 4             |
| 10 Gbps      | 2             |

> 口诀：**带宽越大，开销越小**，因为带宽越高说明路径越“好”。

### 一、选举三步走（按顺序）

**第1步：选根桥**

- 比桥ID（优先级 → MAC），最小者赢
- 本拓扑：SW1 胜出（MAC最小）

**第2步：选根接口（每台非根桥各选1个）**

- 比RPC（到根桥开销），最小者赢
- 本拓扑：SW2 Port1、SW3 Port1 胜出（直连100M，RPC=19）

**第3步：选指定接口（每条链路选1个）**

- 比RPC → 比桥ID → 比端口ID，最小者赢
- 本拓扑：SW2 Port2 胜出（与SW3 RPC相等，但桥ID更小）

**第4步：剩余端口阻塞**

- 本拓扑：SW3 Port2 阻塞，环路打破

------

### 二、本拓扑结构（缩进表示连接）

```
SW1（根桥）
├── 100M（Cost=19）── SW2 Port1（RP）
│                     └── 10M（Cost=100）── SW3 Port2（阻塞）
│
└── 100M（Cost=19）── SW3 Port1（RP）
```



> 说明：SW2和SW3之间通过10M链路互联，SW3端阻塞

------

### 三、最终端口角色清单

- SW1：双端口均为指定接口（DP），根桥默认
- SW2：Port1为根接口（RP），Port2为指定接口（DP）
- SW3：Port1为根接口（RP），Port2为非指定接口（阻塞）

------

### 四、核心口诀

> 根桥看MAC最小
> RP看RPC最小
> DP看完整四步（RPC → 桥ID → 端口ID）
> 剩下端口全阻塞

### 注意

- RPC相同时不能直接得出结论，要继续比桥ID
- 阻塞口仍收BPDU，只是不转发用户数据



## STP端口角色

**根接口（RP，Root Port）**

每台非根桥交换机上有且只有一个根接口，位于去往根桥路径最优的链路上，负责接收来自根桥的BPDU和业务数据。根接口是到达根桥的“最佳入口”，最终状态为转发。

**指定接口（DP，Designated Port）**

每条链路上有且只有一个指定接口，位于该链路上距离根桥最近的交换机上，负责将BPDU和业务数据从根桥方向转发到该链路。根桥的所有端口默认都是指定接口，最终状态为转发。

**非指定接口（Non-DP，Non-Designated Port）**

每条链路上除了指定接口以外的端口都是非指定接口，被STP阻塞以打破环路。非指定接口不转发业务数据，最终状态为阻塞。

**一句话总结**

每台非根桥选一个RP（上行入口），每条链路选一个DP（下行出口），剩下的全部阻塞。

**补充说明**

根桥没有根接口，所有端口都是指定接口

非指定接口是唯一可能被阻塞的角色

只有根接口和指定接口才能进入转发状态

端口角色由STP计算决定，端口状态是计算后的结果

端口角色与端口状态的对应关系：根接口=转发，指定接口=转发，非指定接口=阻塞



## STP端口状态

端口状态按顺序经历五个阶段：禁用 → 阻塞 → 侦听 → 学习 → 转发

**禁用**：端口物理Down或手动关闭，完全不工作，什么都不做。

**阻塞**：端口被STP阻塞以打破环路。不能发BPDU，不能收发业务数据，不能学MAC，唯一能做的是持续侦听BPDU。

**侦听**：端口被初步选为RP或DP，但STP计算未结束。只能收发BPDU，不能收发业务数据，不能学MAC。持续15秒。

**学习**：端口准备进入转发状态前的过渡期。能收业务数据并学MAC（为转发做准备），但不能转发业务数据。持续15秒。

**转发**：端口正常工作，收发业务数据，处理BPDU，学MAC。只有RP或DP才能进入此状态。

补充：阻塞口仍收BPDU，禁用口完全不工作；学习时收数据是为了学MAC，不是正式转发。

## 补充内容

STP端口状态迁移需要满足特定条件：

从阻塞到侦听：端口被选为RP或DP，或阻塞超时（默认20秒，即最大老化时间）

从侦听到学习：等待一个转发延迟（15秒），确保拓扑稳定

从学习到转发：再等待一个转发延迟（15秒），确保无环路

从转发回阻塞：拓扑发生变化，端口角色被重新计算后不再是RP或DP

阻塞口持续收不到BPDU时会触发迁移：阻塞持续20秒收不到BPDU → 进入侦听（15秒）→ 进入学习（15秒）→ 进入转发，总共约50秒的收敛时间

华为设备默认计时器：Hello Time 2秒，Forward Delay 15秒，Max Age 20秒

只有RP和DP能进入转发状态，非指定接口永远停在阻塞状态

## STP计时器

**三个关键计时器**

Hello Time（2秒）：根桥发BPDU的间隔

Forward Delay（15秒）：端口在侦听或学习状态停留的时间

Max Age（20秒）：BPDU的老化时间，超过20秒未收到则重新计算

**收敛时间**

阻塞口等20秒（BPDU老化）→ 侦听15秒 → 学习15秒 → 转发，总计约50秒

如果不用等老化（如管理员手动操作），则是15+15=30秒

边缘端口（接终端）：直接0秒转发，不经过侦听和学习

**一句话总结**

50秒 = 20秒老化 + 15秒侦听 + 15秒学习

**边缘端口补充**

边缘端口默认状态就是转发，它不经过阻塞、侦听、学习这三个状态，一上线就是转发，所以0秒。但如果边缘端口收到了BPDU（说明有人接了交换机），它会立即丧失边缘端口特性，变回普通端口，重新参与STP计算，此时就会经历完整的阻塞→侦听→学习→转发流程。另外，如果开启了BPDU保护，收到BPDU会直接关闭该端口（error-down）并通知管理员。

**根端口故障场景（30秒收敛）**

根端口故障时，阻塞端口一直在侦听BPDU，已同步最新拓扑信息，不需要再等20秒老化确认变化，直接从阻塞→侦听（15秒）→学习（15秒）→转发，耗时30秒。

两种场景对比：根桥整体失效需等20秒老化+15秒侦听+15秒学习=50秒；仅根端口故障时阻塞口已有最新BPDU，省去20秒老化，只需15+15=30秒。
