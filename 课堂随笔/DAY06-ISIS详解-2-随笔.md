# isis详解

isis配置

```
isis 1
	network 49.0001.0000.0000.0001.00
	is-level level-2
	cost-style wide
	auto-cost enable
int g0/0/0
	isis enable
```



ISIS、OSPF类的路由协议 

​	三板斧 

​	（OSPF中的LSA）LSP（泛洪建图）

​	LSDB（收集泛洪的LSP存储）

​	SPF（计算最短路径）

​		 

LSP 采用LSP ID（Link-State Packet ID链路状态报文标识）进行标识，其由三部分组成：

​	系统ID 6字节：产生该LSP的System ID

​	伪节点ID 1字节：只有0和非0的情况，标识其是否为该广播网络的伪节点。

​	分片ID 1字节：一个LSP过大时，其发出的包会进行分片，这个就是分片的标记号。

#### PDU

PDU是工作在二层的，理论上来说，传输的是数据帧



PDU有很多种IIH、LSP、CSNP、PSNP等

他们的报文头部前部分都是相同的，称为基础头部

```
域内路由协议鉴别符（1字节）：固定 0x83，标识IS-IS协议
头部长度（1字节）：PDU头部总字节数，固定为8，因为仅指通用头部（就是这里的八条）。
版本/协议ID扩展（1字节）：固定 1，IS-IS版本号
系统ID长度（1字节）：通常 0，表示System ID占6字节
PDU类型（1字节）：区分报文类型，见下表
版本（1字节）：固定 1
保留（1字节）：固定 0，暂未使用
最大区域地址数（1字节）：设备支持的最大区域地址数量
```



#### IIH（ISIS-Hello）

IIH PDU有三种类型

​	Level-2 IIH（2级设备）

​	Level-1 IIH（1级设备）

​	P2P IIH（p2p 设备）

IIH的构造（只详解特有部分）：

```
基础头部...
+
电路类型（1字节）：L1/L2/L1L2
Source ID（ID+2字节）：发送者 System ID + 00-00
保持时间（2字节）：邻居保留本设备可达的时间
PDU长度（2字节）：报文总长度，通用头部长度+IIH特有+所有TLV的长度，指整个IIH包有多大。
优先级（1字节）：选 DIS 用，仅广播域的报文有，p2p类型网络没有
LAN ID（ID+2字节）：DIS 的 System ID + 伪节点 ID，仅广播域的报文有，p2p类型网络没有
+
TLV部分
```

IIH的发送间隔时间默认为10，Hello包倍数（Hello Multiplier） 默认为3，死亡时间（Dead time）是自动计算的，默认为3x10=30秒

并且和OSPF不同，无需协商hello间隔，各配各的。



#### LSP（Link-State Packet）

 	包含一条或多条链路状态信息，LSP类似于OSPF中的LSA。有三种LSP，通过认证。

LSP的标识非常关键，是三元组：
**系统ID + 伪节点ID + 分片号**。

一个路由器可以有多个LSP，当信息太多时就分片，并用**LSP ID**唯一标识每个分片。

PDU通用头部+LSP载荷：

PDU通用：

​	

```
PDU通用的组成：
基础头部...
+
PDU长度（2字节）：报文总长度，通用头部+LSP特有头部+所有TLV的长度，指整个LSP包有多大。
剩余生存时间（2字节）：老化倒计时，归零后LSP被全网删除。
LSP ID（ID+2字节）：System ID + 伪节点ID + 分片号，唯一标识该LSP。
序列号（4字节）：LSP更新时递增，值越大越新。
校验和（2字节）：校验LSP内容是否损坏。
+
TLV部分
```

​	

LSP载荷：LSP特有的头部+TLV（可变长字段，可以支持扩展，例如可以额外支持IPV6）

​	LSP特有头部：

​			PDU长度（PDU Length）：指示该PDU的总长度（字节）

​			剩余生存时间（Remaining LifeTime）

​			



#### CSNP：

​	包含LSDB所有的LSP的摘要信息，包含4个重要信息：

- LSP ID
- 序列
- 剩余生存时间
- 校验和

​	CSNP会由DIS定期发送（发送周期缺省为10秒）用于其他设备检查LSDB中信息是否缺失或更新，有就进行更新。

​	P2P网络只有第一次建立会发送CSNP，因为该网络只会有两端的设备，不会有新增或删除路由。

CSNP的组成：

```
基础头部...
+
Source ID（ID 6+2=8字节）：发送者 System ID + 00-00
Start LSP ID（ID 6+2=8字节）：摘要范围的起始LSP ID
End LSP ID（ID 6+2=8字节）：摘要范围的结束LSP ID
+
TLV部分（每条LSP摘要条目）：
剩余生存时间（2字节）
LSP ID（ID 6+2=8字节）
序列号（4字节）
校验和（2字节）
```

#### PSNP（Partial Sequence Number部分序列号报文）：

​	包含LSP的部分摘要信息，主要用于请求部分LSP更新，还用于P2P网络中的LSP。（类似于LSR和LSAck）

​	只有在出现路由

PSNP的组成：

```
基础头部（8字节）
+
Source ID（ID 6+2=8字节）：发送者 System ID + 00-00
Start LSP ID（ID 6+2=8字节）：请求/确认范围的起始LSP ID
End LSP ID（ID 6+2=8字节）：请求/确认范围的结束LSP ID
+
TLV部分（结构同CSNP，只放请求或确认的那几条）
```

ISIS中的路由渗透：

​	Level 1-2中可以把部分的L2路由以L1 LSP的形式泄露到L1区域，这样L1设备就能得知这些明细路由，可以计算最优路径。



#### 部分总结

IIH：用来进行链路上发现和建立邻居

LSP：用来发送全量或部分请求的LSP的具体信息（LSP是报文也是信息条目）

#### IS-IS PDU的组成

IS-IS PDU由三部分组成：

​	通用头部：固定8字节

​	PUD特有头部：和PDU类型相关，如IIH、LSP、SNP都有各自的特有头部部分

​	LV 部分：可变长，承载具体信息（邻居、路由、认证等），不同 PDU 类型携带不同 TLV



#### TLV

TLV的核心作用可以灵活、可扩展的方式协议数据

特点：   灵活扩展：无需修改协议头部就可以支持新功能（如IPv6、流量工程）

​	动态适配：不同的数据类型可以通过独立的TLV封装，互不干扰。

​	高效解析与兼容性：接收方通过Type字段快速识别TLV

TLV每个元素如下：

​	类型Type：1Byte，标识类型，IS-IS定义了丰富的类型。

​	长度：1Byte，标识TLV中的value的字段长度。

​	值value：可变字段，长度由Length字段描述。



#### DIS与伪节点

​	在广播区域内会选一个作为DIS，可以查看到在设备条目中看到多了一个倒数第二个字节非0的设备，其就为的DIS，是一个虚拟设备，相当于该设备的第二身份。

​	

​	作用：	

​		1.减少广播域内所有节点整体发送的LSP数量。

​		2.默认一台路由器必须用LSP描述一段链路中该路由器所有邻居关系。

​		3.路由器仅需描述字节和DIS的邻居关系

​		4.没有减少邻居关系。

​	

**DIS 的选举：**

- 优先级默认为 **64**，取值范围 **0-127**，值越大越优先，**0 也表示参与选举**（与 OSPF 不同，OSPF 中 0 不参与）
- 配置命令：`isis dis-priority <0-127>`，在接口视图下配置
- 选举规则：
  - 优先级高的当选
  - 优先级相同时，**接口 MAC 地址大的当选**（不是 System ID）
  - 优先级全为 0 时，也会选举出 DIS，仍按 MAC 地址比大小
- DIS 支持**抢占**，一旦出现更高优先级或同优先级 MAC 更大的路由器，立即抢占成为新 DIS
- IS-IS **没有备份 DIS**，DIS 故障后直接重新选举
- Level-1 和 Level-2 的 DIS **分别独立选举**，可以是同一台，也可以不同
- DIS 每 **10 秒** 发送一次 CSNP，广播网络上 **Hello 间隔默认为 10 秒**，但 DIS 的 Hello 间隔是 **3.3 秒（10/3）**，比普通路由器快三倍，用于快速检测故障
- 广播网络上**所有路由器之间两两建立邻接关系**，DIS 不减少邻接数，邻接数仍为 N×(N-1)/2
- DIS 的作用是：
  - 生成**伪节点 LSP**（代表广播网络）
  - 定期发送 **CSNP** 做数据库对账



#### ISIS的邻居建立

​	通过Hello报文进行建立连接（三次握手），完成后会发送全量信息。

​	广播网络的确认方式是静默确认，不发报文就是确认。

​	通过`display isis peer`可以查看Curcuit ID字段看到DIS的信息，确认谁是DIS

​	`display isis lsdb verbose`可以查看到区域、本地的接口、邻居等信息



#### ISIS的邻居信息同步过程

AR1和AR2同步过程（广播）：

​	1.AR1和AR2到达UP状态，双方立刻互发发送全量LSP

​	2.双方不发送信息，静默方式确认（P2P用发PSNP确认）

​	3.DIS选举完成，由DIS发送描述当前广播网络拓扑的LSP

​	如下：

```
0000.0000.0001.10-00
0000.0000.0001.00-00
0000.0000.0002.00-00
```

​	4.当拓扑发生变化，LSP信息过期或缺失，对应路由器主动发送PSNP（组播）

​	5.DIS每10秒发送一次CSNP（组播），有增量会自动发送LSP

当拓扑发生变化，如AR3加入

​	1.AR3会发送一次LSDB内全量的LSP

​	2.AR3查看DIS发送的CSNP，发现缺少信息，发送请求PSNP

​	3.信息对应的路由器收到后发送LSP告知信息。

1. **AR1 和 AR2 同步过程（P2P）：**

   1. AR1 和 AR2 到达 UP 状态，双方立刻互发全量 LSP

   2. 收到 LSP 的一方必须回复 PSNP 作为显式确认

   3. 双方不选举 DIS，无伪节点 LSP，各自 LSP 直接描述 P2P 链路

      如下：

      ```
      0000.0000.0001.00-00
      0000.0000.0002.00-00
      ```

   4. 当拓扑发生变化，LSP 信息过期或缺失，对应路由器发送 PSNP 请求更新

   5. 邻接建立时双方互发一次 CSNP 做初始同步，后续有新 LSP 直接发送，不再周期性发送 CSNP

   

**AR1 和 AR2 同步过程（P2P）：**

1. AR1 和 AR2 到达 UP 状态，双方立刻互发全量 LSP

2. 接收方收到一条 LSP 即回复一条 PSNP 确认，逐条发送逐条确认；若发送方同时有多条 LSP 更新，则逐条发送，接收方逐条回复 PSNP

3. 双方不选举 DIS，无伪节点 LSP，各自 LSP 直接描述 P2P 链路

   如下：

   

   ```
   0000.0000.0001.00-00
   0000.0000.0002.00-00
   ```

   

4. 当拓扑发生变化，LSP 信息过期或缺失，对应路由器发送 PSNP 请求更新

5. 邻接建立时双方互发一次 CSNP 做初始同步，后续有新 LSP 直接发送，不再周期性发送 CSNP；接收方收到 LSP 后逐条回复 PSNP 确认

```
控制平面：

邻居建立：通过 IIH（Hello）报文发现和维护邻居关系，选举 DIS

数据库同步：通过 CSNP/PSNP 完成 LSDB 同步，确保全网拓扑一致

路由计算：基于 LSDB 运行 SPF 算法，独立计算最短路径，生成路由表

转发平面：

路由表下发：将计算出的最优路由下发到转发信息库（FIB）

报文转发：根据 FIB 查找出接口和下一跳，封装数据链路层头直接转发
```
