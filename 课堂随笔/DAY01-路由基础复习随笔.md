HCIA阶段最重要的是 路由/交换 --> 通信

分为三层通信和二层通信

二层解决同一链路下的通信，三层解决不同链路下的通信

数据包

​	源/目MAC地址 （二层）

​	源/目IP地址  （三层）

路由 查表转发（路由表） 单波

​	数据包转发通过查表，找到对应的表条目

​	路由表条目： 目标网段/掩码 下一跳 出接口

交换 查表转发

​	二层交换机只有MAC地址表	

​	转发可以单播、广播

​	目标为全F的MAC --> 实现泛洪



所有的路由协议都是为了实现端到端通信的

静态路由

​	普通下一跳为什么不能指定接口

​	因为数据通过转发时，只指定了出接口，所以路由会默认认为目标接口是在当前的mac地址，只会在本地的反复请求目标地址的mac，没人回应，没有目标mac无法封装，导致出现链路不通的问题

​	指定出接口为下一跳，会以出接口IP作为下一跳路由地址进行填充

​	

```
int g0/0/0
	ip address 10.0.12.1 24
ip route-static 2.2.2.2 32 g0/0/0

会直接等价于 ip route-static 2.2.2.2 32 10.0.12.1
查询也是这样显示
```

​	只需要在路由上开启arp代理，就可以实现下一跳 指定出接口实现路由通信
