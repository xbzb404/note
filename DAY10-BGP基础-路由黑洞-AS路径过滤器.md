# DAY10：BGP基础-路由黑洞-AS路径过滤器

## 一、AS_Path路径属性

### 1.1 基本概念
- AS_Path是经过路由的AS区域列表
- **排列规则**：最早经过的在最右边，依次在开头加入
- **示例**：`AS_Path: 200 100` → 100是始发AS号，200是最近经过的AS

### 1.2 AS_Path的三个作用

| 作用                   | 说明                                          |
| ---------------------- | --------------------------------------------- |
| **EBGP防环**           | 收到包含自身AS号的路由时直接丢弃              |
| **路由过滤与路径控制** | 通过匹配AS_PATH中的AS号控制路由收发及路径长短 |
| **选路参考**           | BGP选路规则中路径越短越优                     |

### 1.3 AS_Path类型

| 类型                    | 说明                                 | 示例             |
| ----------------------- | ------------------------------------ | ---------------- |
| **AS_SEQUENCE**（默认） | 有序列表，从左到右最近→最远          | `200 100 400`    |
| **AS_SET**（聚合时）    | 无序集合，被聚合子路由的AS号放入`{}` | `300 {100, 200}` |

### 1.4 AS_Path在报文中的格式（TLV）

```
BGP UPDATE
  └─ Path Attributes
      └─ AS_PATH (Type 2)
          ├─ Flags: 0x40 (公认必遵)
          ├─ Length: 6
          └─ Value
              ├─ Segment Type: AS_SEQUENCE
              ├─ Segment Length: 3
              ├─ AS 100  ← 最近（转发该路由给本机的区域）
              ├─ AS 300
              └─ AS 200  ← 最远（路由始发区域）
```

### 1.5 AS-Path Filter（AS路径过滤器）

**定义**：基于AS_Path内容进行路由过滤或策略控制的工具

**使用方法**：
1. 定义过滤器：`ip as-path-filter {name | number} {permit | deny} regex`
2. 在Route-Policy中调用：`if-match as-path-filter 1`
3. 在BGP邻居配置中调用：`peer x.x.x.x filter-policy as-path-filter import`

**常用正则表达式**：

| 元字符 | 含义           | 示例       | 匹配说明            |
| ------ | -------------- | ---------- | ------------------- |
| `^`    | 字符串开始     | `^100`     | 路径以100开头       |
| `$`    | 字符串结束     | `100$`     | 路径以100结尾       |
| `_`    | AS号边界       | `_100_`    | 匹配独立AS号100     |
| `.`    | 任意单个字符   | `1.0`      | 匹配110、120等      |
| `*`    | 0次或多次      | `100*`     | 匹配10、100、1000等 |
| `+`    | 1次或多次      | `100+`     | 匹配100、1000等     |
| `\|`   | 逻辑或         | `100\|200` | 匹配100或200        |
| `[ ]`  | 括号内任意字符 | `[12]00`   | 匹配100或200        |
| `( )`  | 分组           | `(100)+`   | 匹配100、100100等   |

**常用正则技巧**：
- 匹配空路径（本地始发）：`^$`
- 匹配包含AS 100：`_100_`
- 匹配路径长度为1：`^[0-9]+$`

**过滤示例**：
```
ip as-path-filter denyAs100 deny _100_    # 拒绝包含AS100的路由
ip as-path-filter denyAs100 permit .*      # 放行其他路由
bgp 100
 peer 1.1.1.1 as-path-filter denyAs100 import
```

## 二、BGP路由注入方式

### 2.1 三种注入方式

| 方式             | 特点                                                         |
| ---------------- | ------------------------------------------------------------ |
| **Network**      | 逐条宣告，必须存在于本地路由表中；路径属性略优于import-route |
| **Import-route** | 批量引入（OSPF、ISIS等）；路径属性略逊于network              |
| **聚合路由**     | 路由汇总，可减少路由表条目                                   |

### 2.2 核心原则
> **BGP不生产路由，只搬运路由**
> - Network宣告的路由**必须**已在本地路由表中存在
> - IGP从零到有生成路由，BGP仅传递路由

---

## 三、BGP路由通告原则

1. **只发布最优（Best）且有效（Valid）的路由**
2. **从EBGP获得的路由会发布给所有对等体**
3. **从IBGP获得的路由不会发布给其他IBGP对等体**（水平分割）

---

## 四、水平分割与解决方案

### 4.1 水平分割原则
> 从IBGP对等体获取的BGP路由，不会再发送给其他IBGP对等体

### 4.2 解决方案对比

| 方案                      | 说明                         | 现状                 |
| ------------------------- | ---------------------------- | -------------------- |
| **全互联**                | 每台设备建立n-1个邻居        | 基本不用（过于繁琐） |
| **路由反射器（RR）**      | 客户端路由反射给所有邻居     | **推荐使用**         |
| **联邦（Confederation）** | 大AS中划分小AS，联邦EBGP互通 | 基本不用（配置复杂） |

### 4.3 路由反射器（RR）规则

| 收到路由的来源 | 反射给客户端 |    反射给非客户端     |
| -------------- | :----------: | :-------------------: |
| **客户端**     |      ✅       |           ✅           |
| **非客户端**   |      ✅       | ❌（遵循IBGP水平分割） |

### 4.4 联邦（Confederation）
- 在大AS中划分小AS，小区域间形成联邦EBGP对等体
- 联邦EBGP内部仍可使用全互联或RR解决水平分割
- 联邦EBGP使用联邦AS号写入AS_Path解决环路问题

## 五、路由黑洞与同步规则

### 5.1 路由黑洞成因
```
R1(EBGP) --- R3（IBGP） --- R4（互联路由、未运行BGP） --- R6（IBGP） --- R2(EBGP)
```
- IBGP邻居通过非BGP区域建立虚拟连接
- 中间设备未运行BGP，不识别BGP路由
- 导致EBGP路由在中间节点被丢弃

### 5.2 同步规则（Synchronization）
> 从IBGP学习到路由时，必须在IGP协议中同时学到该路由，才能转发给EBGP邻居

**工作流程**：
1. R3从R1的IBGP学到10.0.0.0/8
2. 开启同步规则后，R3检查IGP（如ISIS）是否有该路由
3. 若IGP中无该路由，R3**不允许转发**给R7
4. 只有将路由引入IGP后，才能转发给EBGP邻居

> **现状**：默认关闭（`no synchronization`），实际生产中基本不使用

**开启命令**：`synchronization`

### 5.3 路由引入顺序
> **先引入IGP，再从IGP引入BGP**，避免路由引入无效

## 六、BGP基础配置补充

### **6.1 Router-ID**

- 可手动设置（**强烈推荐**）
- 自动选择顺序：**手动配置 > 最大的Loopback接口地址 > 最大的物理接口地址**

### **6.2 EBGP TTL**

- 默认TTL=1（推荐使用直连接口建立）
- 使用回环口或跨设备建立时需修改，命令例如：`peer 2.2.2.2 ebgp-max-hop 255`，**同时需配置connect-interface指定源接口，并确保路由可达**

## 七、路径属性分类

### 7.1 分类总览

```
路径属性
├── 公认（所有路由器可识别）
│   ├── 公认必遵（Update必须携带）
│   │   ├── AS_Path
│   │   ├── Origin
│   │   └── Next_Hop
│   └── 公认任意（非必须携带）
│       └── Local_Pref
└── 可选（不要求所有路由器识别）
    ├── 可选过渡（不认识也接受并传递）
    │   └── Community
    └── 可选非过渡（不认识则忽略，不传递）
        └── MED
```

### 7.2 常用路径属性速查

| 属性           | 类别       | 说明                                 |
| -------------- | ---------- | ------------------------------------ |
| **AS_Path**    | 公认必遵   | 防环+选路（越短越优）                |
| **Origin**     | 公认必遵   | 路由来源（IGP/EGP/incomplete）       |
| **Next_Hop**   | 公认必遵   | 下一跳地址                           |
| **Local_Pref** | 公认任意   | 本地优先级（越大越优，仅IBGP传递）   |
| **MED**        | 可选非过渡 | 多出口区分（越小越优，仅传递给EBGP） |
| **Community**  | 可选过渡   | 团体属性（批量路由标记）             |
