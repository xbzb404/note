# STP/RSTP/MSTP 协议笔记（Day 15）

---

## 一、协议演进与兼容性

| 协议               | 特点                                           | 兼容性                |
| ------------------ | ---------------------------------------------- | --------------------- |
| **STP**（802.1D）  | 早期协议，收敛慢（30~50秒）                    | —                     |
| **RSTP**（802.1W） | 快速收敛，增加边缘端口、Proposal/Agreement机制 | 可兼容STP             |
| **MSTP**（802.1S） | 多实例生成树，可实现VLAN级负载均衡             | **同时兼容STP和RSTP** |

> ⚠️ **注意**：华为交换机默认运行MSTP，接入仅支持STP的老设备时，MSTP会自动将对应端口降级为STP模式（通过BPDU版本号识别适配）。

---

## 二、根桥（Root Bridge）

- **定义**：STP树拓扑的起点，**桥ID最小**的设备。
- **作用**：所有端口角色（RP/DP/阻塞）均以**到达根桥的路径开销**为选举依据。

> **举例**：三台交换机优先级均为默认32768，MAC最小的成为根桥。

---

## 三、桥ID（Bridge ID）

| 字段     | 长度   | 默认值    | 说明                       |
| -------- | ------ | --------- | -------------------------- |
| 桥优先级 | 16 bit | **32768** | 可手动修改（步长4096）     |
| MAC地址  | 48 bit | 设备自带  | 优先级相同时作为次选比较项 |

> **选举规则**：**先比优先级（数值越小越优）**，再比MAC（数值越小越优）。

---

## 四、STP接口Cost值与RPC

- **Cost值**：端口带宽对应的路径开销，不同标准下值不同。
- **修改命令**（华为）：`stp pathcost-standard { dot1d-1998 | dot1t | legacy }`（需全网统一）
- **RPC**：从本交换机到根桥的**所有入方向端口Cost累加和**。

**常用参考值（dot1t标准）**：

| 带宽     | Cost值    |
| -------- | --------- |
| 10 Mbps  | 2,000,000 |
| 100 Mbps | 200,000   |
| 1 Gbps   | 20,000    |
| 10 Gbps  | 2,000     |

> **举例**：根桥SW1 —(1G, cost=20000)— SW2 —(100M, cost=200000)— SW3，SW3的RPC = 20000 + 200000 = **220000**。

---

## 五、接口ID（Port ID）

- 组成：**端口优先级（4 bit）+ 端口编号（12 bit）**，默认端口优先级为 **128**。
- 仅在 **RPC相同** 时作为**第三位比较项**（前两位：桥ID → RPC → 接口ID）。

---

## 六、端口角色（Port Roles）

| 角色                       | 位置       | 作用                        |
| -------------------------- | ---------- | --------------------------- |
| **根端口（RP）**           | 非根桥上   | RPC最小的端口，负责接收BPDU |
| **指定端口（DP）**         | 每段链路上 | RPC较小的一端，负责发送BPDU |
| **阻塞端口（Blocked/AP）** | 冗余链路   | 被阻塞，仅侦听BPDU          |

> **选举口诀**：全网选根桥 → 每台非根桥选RP（RPC最小）→ 每条链路选DP（RPC较小端）→ 剩余全阻塞。

## 七、BPDU部分重点

### 7.1 两种BPDU对比

|              | **配置BPDU**                            | **TCN BPDU**                  |
| ------------ | --------------------------------------- | ----------------------------- |
| **谁发**     | 根桥周期性发（默认2秒）；非根桥转发     | 非根桥检测到拓扑变化时发      |
| **方向**     | 从根桥向下游泛洪                        | 从变化点向根桥方向上传        |
| **核心作用** | 维护拓扑，传递选举信息                  | 仅作为"拓扑变化通知"信号      |
| **内容**     | 携带根桥ID、RPC、桥ID、端口ID等完整信息 | **仅4字节**，不含任何选举信息 |

### 7.2 Flags标志位（RSTP提速的关键）

RSTP中Flags字段的8个bit全部利用：

- **Bit 6（Proposal）** + **Bit 3（Agreement）**：上下游端口快速协商，**无需等待Forward Delay**，实现毫秒级收敛。
- **Bit 7（TC）**：拓扑变化标志，触发全网MAC表快速老化。
- **Bit 0（TCA）**：对TCN的确认回复。

### 7.3 TCN交互流程（仅需记住4步）

> **检测到变化 → 向上游发TCN → 上游回TCA确认 → 根桥收到后全网发TC置位BPDU → 15秒内MAC表老化刷新**

### 7.4 ⚠️ BPDU注意事项

1. **BPDU的目的MAC是组播**：`01-80-C2-00-00-00`（不经过IP层转发）。
2. **非根桥不生成配置BPDU**，而是修改（Bridge ID、Port ID、RPC、Message Age）后**转发**根桥的BPDU。
3. **Message Age**从根桥发出为0，每跳+1，超过**Max Age（20秒）**未收到新BPDU则认为链路失效。
4. **计时器（Hello/Max Age/Forward Delay）仅在根桥上修改有效**，非根桥设置被覆盖。
5. **MAC表老化**：正常300秒，收到TC置位BPDU后缩短为**Forward Delay（15秒）**，防止数据被错误转发到旧路径。
6. **安全防护**：边缘端口建议开启 **BPDU Guard**（收到BPDU即err-down，防止非法交换机接入）。

---

## 八、补充重点总结

1. **STP端口状态迁移**：Blocking（20秒）→ Listening（15秒）→ Learning（15秒）→ Forwarding（共30~50秒）
2. **RSTP改进**：边缘端口（连接终端直接转发）+ Proposal/Agreement快速协商
3. **MSTP多实例**：不同VLAN映射不同实例，各自独立计算生成树，实现**负载均衡**（例：VLAN 10走SW1根桥，VLAN 20走SW2根桥）

---

**📌 总口诀**：STP选路靠三级（桥ID→RPC→端口ID），配置BPDU保心跳，TCN报警刷MAC，RSTP提速靠Flag，MSTP多实例做负载。
