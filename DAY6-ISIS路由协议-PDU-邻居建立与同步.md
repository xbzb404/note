# DAY6：ISIS路由协议-PDU解析-邻居建立与同步

### 一、基本概念

IS-IS 属于链路状态路由协议，三板斧：

1. **泛洪建图**：通过 LSP 泛洪链路状态信息
2. **收集存储**：所有路由器收集 LSP，形成 LSDB
3. **SPF 计算**：以自己为根，运行 Dijkstra 算法计算最短路径

---

### 二、LSP 标识

LSP 由三元组唯一标识，称为 **LSP ID**：

| 组成      | 长度   | 说明                                         |
| :-------- | :----- | :------------------------------------------- |
| 系统 ID   | 6 字节 | 产生该 LSP 的路由器 System ID                |
| 伪节点 ID | 1 字节 | `00` = 普通 LSP，非 0 = DIS 产生的伪节点 LSP |
| 分片号    | 1 字节 | LSP 过大时分片，从 0 开始编号                |

**注意：LSP 既是报文类型，也是 LSDB 中的信息条目。**

---

### 三、PDU 总览

IS-IS 直接封装在数据链路层，PDU 是二层帧。

#### PDU 通用头部（8 字节）

| 字段               | 长度   | 说明                         |
| :----------------- | :----- | :--------------------------- |
| 域内路由协议鉴别符 | 1 字节 | 固定 `0x83`，标识 IS-IS 协议 |
| 头部长度           | 1 字节 | 固定 `8`，仅指通用头部       |
| 版本/协议 ID 扩展  | 1 字节 | 固定 `1`                     |
| 系统 ID 长度       | 1 字节 | 通常 `0`，表示 6 字节        |
| PDU 类型           | 1 字节 | 区分报文类型                 |
| 版本               | 1 字节 | 固定 `1`                     |
| 保留               | 1 字节 | 固定 `0`                     |
| 最大区域地址数     | 1 字节 | 设备支持的最大区域地址数     |

#### PDU 类型速查

| 值   | 报文       |
| :--- | :--------- |
| 15   | L1 LAN IIH |
| 16   | L2 LAN IIH |
| 17   | P2P IIH    |
| 18   | L1 LSP     |
| 20   | L2 LSP     |
| 24   | L1 CSNP    |
| 25   | L2 CSNP    |
| 26   | L1 PSNP    |
| 27   | L2 PSNP    |

#### PDU 结构

```
┌──────────────┐
│  通用头部     │  固定8 字节
├──────────────┤
│  特有头部     │  因 PDU 类型不同
├──────────────┤
│  TLV 部分    │  可变长
└──────────────┘
```

---

### 四、IIH（IS-IS Hello）

用于发现和建立邻居。

#### IIH 类型

- L1 LAN IIH（PDU 类型 15）
- L2 LAN IIH（PDU 类型 16）
- P2P IIH（PDU 类型 17）

#### IIH 特有字段

| 字段      | 长度   | 说明                                               |
| :-------- | :----- | :------------------------------------------------- |
| 电路类型  | 1 字节 | `1`=L1，`2`=L2，`3`=L1L2                           |
| Source ID | 8 字节 | 发送者 System ID + `00-00`                         |
| 保持时间  | 2 字节 | 邻居保留本设备可达的最大秒数                       |
| PDU 长度  | 2 字节 | 整个 IIH 包的总长度                                |
| 优先级    | 1 字节 | 选 DIS 用，仅广播域有，P2P 没有                    |
| LAN ID    | 8 字节 | DIS 的 System ID + 伪节点 ID，仅广播域有，P2P 没有 |

**TLV 部分**：区域地址、IS 邻居（MAC 地址列表）、填充（到 MTU）、认证、三次握手（P2P）

#### IIH 计时器

- 发送间隔默认 **10 秒**
- Hello 倍数默认 **3**
- 死亡时间自动计算 = **10 × 3 = 30 秒**
- **无需协商**，各配各的，与 OSPF 不同

---

### 五、LSP（链路状态 PDU）

承载链路状态信息，类似 OSPF 的 LSA。

#### LSP 特有字段

| 字段         | 长度   | 说明                       |
| :----------- | :----- | :------------------------- |
| PDU 长度     | 2 字节 | 整个 LSP 包总长度          |
| 剩余生存时间 | 2 字节 | 老化倒计时，归零后全网删除 |
| LSP ID       | 8 字节 | 三元组，唯一标识该 LSP     |
| 序列号       | 4 字节 | 越大越新                   |
| 校验和       | 2 字节 | 校验 LSP 内容完整性        |

**TLV 部分**：区域地址、IS 邻居（System ID + 开销）、IP 内部可达、IP 外部可达、认证等

---

### 六、CSNP（完全序列号 PDU）

包含 LSDB 中**所有 LSP 的摘要**，用于数据库对账。

#### CSNP 特有字段

| 字段         | 长度   | 说明                       |
| :----------- | :----- | :------------------------- |
| Source ID    | 8 字节 | 发送者 System ID + `00-00` |
| Start LSP ID | 8 字节 | 摘要范围起始               |
| End LSP ID   | 8 字节 | 摘要范围结束               |

**TLV：LSP Entries**（L1 类型=`9`，L2 类型=`11`），每条摘要条目 16 字节：

| 字段         | 长度   |
| :----------- | :----- |
| 剩余生存时间 | 2 字节 |
| LSP ID       | 8 字节 |
| 序列号       | 4 字节 |
| 校验和       | 2 字节 |

#### CSNP 发送规则

- **广播网络**：DIS 每 **10 秒** 周期发送
- **P2P 网络**：**仅邻接建立时发送一次**，后续不再发送（因为只有两端，无新增设备）

---

### 七、PSNP（部分序列号 PDU）

包含**部分 LSP 摘要**，用于请求缺失 LSP 或确认收到 LSP。

#### PSNP 特有字段

结构同 CSNP，但 LSP Entries 只放请求或确认的那几条。

#### PSNP 作用

- 请求缺失或过时的 LSP（类似 OSPF 的 LSR）
- P2P 网络中确认收到的 LSP（类似 OSPF 的 LSAck）

---

### 八、TLV

**核心作用：** 灵活、可扩展地承载协议数据。

**特点：**

- 灵活扩展：无需修改协议头部即可支持新功能（如 IPv6、流量工程）
- 动态适配：不同数据类型通过独立 TLV 封装，互不干扰
- 高效解析：接收方通过 Type 字段快速识别

**TLV 结构：**

| 字段   | 长度   | 说明               |
| :----- | :----- | :----------------- |
| Type   | 1 字节 | 标识类型           |
| Length | 1 字节 | Value 字段的字节数 |
| Value  | 变长   | 实际数据           |

---

### 九、DIS 与伪节点

#### 伪节点

广播网络中 DIS 产生的虚拟设备，LSP ID 中伪节点 ID **非 0**，相当于 DIS 的第二身份。

**`display isis lsdb verbose`** 可看到伪节点 LSP。

#### DIS 作用

1. 生成伪节点 LSP，描述广播网络拓扑
2. 每 10 秒发送 CSNP 做数据库对账
3. 路由器仅需描述自己与 DIS 的邻居关系
4. **不减少邻居关系数**，广播网上所有路由器仍两两建立邻接

#### DIS 选举规则

- 优先级默认 **64**，范围 **0-127**，值越大越优先
- **0 也参与选举**（与 OSPF 不同）
- 优先级相同则 **接口 MAC 地址大** 的当选（不是 System ID）
- 优先级全 0 也会选出 DIS
- **支持抢占**，出现更高优先级立即抢（与 OSPF DR 不同）
- **无备份 DIS**，故障直接重新选举
- L1 和 L2 的 DIS **分别独立选举**
- DIS 的 Hello 间隔为 **3.3 秒**（10/3），比普通路由器快三倍，用于快速检测故障

#### 查看命令

```
display isis peer       // Circuit ID 字段可看到 DIS 信息
display isis interface verbose  // 查看 DIS 优先级和选举结果
```

#### 配置命令

```
[Huawei-GigabitEthernet0/0/1] isis dis-priority 120
```

### 十、邻居建立

通过 IIH **三次握手** 建立邻接，完成后互发全量 LSP。

**广播网络确认方式：** 静默确认，不发报文即表示确认（与 P2P 不同）

---

### 十一、邻居信息同步过程

#### 广播网络（如 AR1 和 AR2）

1. AR1 和 AR2 到达 UP 状态，双方立刻互发全量 LSP
2. 双方不发送确认，采用**静默确认**
3. DIS 选举完成，DIS 发送描述广播网络拓扑的**伪节点 LSP**，例如：
   ```
   0000.0000.0001.10-00   // DIS 产生的伪节点 LSP
   0000.0000.0001.00-00   // AR1 自身 LSP
   0000.0000.0002.00-00   // AR2 自身 LSP
   ```
4. 拓扑变化、LSP 过期或缺失时，对应路由器主动发送 **PSNP**（组播）请求更新
5. DIS 每 **10 秒** 发送一次 **CSNP**（组播），有增量时自动发送 LSP

#### AR3 加入广播网络

1. AR3 发送自己 LSDB 内**全量 LSP**
2. AR3 收到 DIS 的 CSNP，对比发现缺失其他路由器的 LSP，发送 **PSNP** 请求
3. 拥有该 LSP 的路由器收到后，发送 LSP 告知 AR3

#### P2P 网络（如 AR1 和 AR2）

1. AR1 和 AR2 到达 UP 状态，双方立刻互发全量 LSP
2. 接收方收到一条 LSP 即回复一条 PSNP 确认，**逐条发送逐条确认**；若同时有多条 LSP 更新，则逐条发送，接收方逐条回复
3. 双方**不选举 DIS**，无伪节点 LSP，各自 LSP 直接描述 P2P 链路：
   ```
   0000.0000.0001.00-00
   0000.0000.0002.00-00
   ```
4. 拓扑变化、LSP 过期或缺失时，对应路由器发送 PSNP 请求更新
5. 邻接建立时双方互发一次 CSNP 做初始同步，后续有新 LSP 直接发送，不再周期性发送 CSNP；接收方收到 LSP 后仍逐条回复 PSNP 确认

---

### 十二、路由渗透

L1-2 路由器可以将部分 L2 路由以 **L1 LSP** 形式泄露到 L1 区域。

**作用：** 让 L1 设备获知明细路由，计算最优路径，避免次优转发。

---

### 十三、配置示例

```
isis 1
  network 49.0001.0000.0000.0001.00
  is-level level-2
  cost-style wide
  auto-cost enable

interface GigabitEthernet0/0/0
  isis enable
```
