# DAY7：ISIS 路由协议-默认路由的控制-路由渗透-路由认证

## 一、区域与路由器类型

### 1.1 骨干区域与非骨干区域

| 区域类型       | 组成                             | 区域号要求 |
| :------------- | :------------------------------- | :--------- |
| **骨干区域**   | 连续的 Level-2 或 Level-1-2 设备 | 可以不同   |
| **非骨干区域** | 区域号相同的 Level-1 设备        | 必须相同   |

> 骨干区域不一定有固定的区域号，只要 Level-2 设备连续连接就形成骨干。

### 1.2 路由器类型

| 类型          | 说明                                              | LSDB 数量       |
| :------------ | :------------------------------------------------ | :-------------- |
| **Level-1**   | 只属于一个非骨干区域，只与本区域 L1/L1-2 建立邻接 | 1 个（L1）      |
| **Level-2**   | 属于骨干区域，只与 L2/L1-2 建立邻接               | 1 个（L2）      |
| **Level-1-2** | 同时属于非骨干区域和骨干区域，是区域边界          | 2 个（L1 + L2） |

## 二、默认路由与 ATT 位

### 2.1 机制说明

当 **Level-1-2 路由器** 与 Level-2 设备相连时，它会在自己发送的 **Level-1 LSP** 中将 ATT 位置 1。收到该 LSP 的 **Level-1 路由器** 会**自动生成一条指向该 L1-2 路由器的默认路由**。

- 作用：引导 L1 路由器将跨区域流量通过 L1-2 转发到 L2 骨干
- 方向：**单向**（L1-2 → L1），不会反向传播

### 2.2 手动控制 ATT 位

**在 Level-1-2 上强制开启/关闭 ATT：**

```
isis 1
 attached-bit advertise always    # 强制 ATT=1，L1 会生成默认路由
 attached-bit advertise never     # 强制 ATT=0，L1 不会生成默认路由
```

> 警告：同一区域内多台设备配置 `always` 可能引发路由环路。

**在 Level-1 上拒绝学习默认路由：**

```
isis 1
 attached-bit avoid-learning       # 即使收到 ATT=1 也不生成默认路由
```



### 2.3 发布默认路由（default-route-advertise）

| 命令                                            | 配置位置    | 作用                                    |
| :---------------------------------------------- | :---------- | :-------------------------------------- |
| `default-route-advertise level-1`               | L1-2 路由器 | 向 L1 区域发布默认路由                  |
| `default-route-advertise always avoid-learning` | 任意路由器  | 发布默认路由 + 自己不学习别人的默认路由 |

## 三、路由渗透（Route Leaking）

### 3.1 什么是路由渗透？

**默认情况下**：L1-2 路由器只向 L1 区域发送**默认路由**，不发送 L2 区域或其他 L1 区域的**明细路由**。

**路由渗透的作用**：把 L2 区域（或其他 L1 区域）的**明细路由**主动引入到本地 L1 区域，让 L1 路由器能看到更详细的路由信息。

### 3.2 基本命令

```
isis 1
 import-route isis level-2 into level-1
```



### 3.3 配合路由策略使用

在 Level-1-2 路由器上，只将 Level-2 区域中的 `10.2.0.0/16` 网段（包括其子网）渗透到 Level-1 区域，其他 Level-2 路由不向下渗透。

```
ip ip-prefix allow-L2toL1 index 10 permit 10.2.0.0 16 # 创建一个叫allow-L2toL1的前缀匹配列表用于过滤10.2.0.0/16

route-policy L2toL1 permit node 10 #创建策略L2toL1，默认匹配即通过
 if-match ip-prefix allow-L2toL1 # 把前缀匹配列表应用到if-match

isis 1
 import-route isis level-2 into level-1 route-policy L2toL1  
 #使用route-policy参数把策略应用到引入中，可以将指定的路由引入到L1区域，避免L1区域的路由表过于臃肿
```



### 3.4 渗透方向总结

| 方向              | 默认行为               | 可配置行为             |
| :---------------- | :--------------------- | :--------------------- |
| Level-1 → Level-2 | 自动（明细路由）       | 可用路由策略过滤       |
| Level-2 → Level-1 | 只有默认路由（ATT 位） | 可通过路由渗透引入明细 |

## 四、路由认证

### 4.1 三种认证类型

| 认证类型       | 配置位置      | 保护范围                     |
| :------------- | :------------ | :--------------------------- |
| **接口认证**   | 接口视图      | 该接口发送/接收的 Hello 报文 |
| **区域认证**   | ISIS 进程视图 | Level-1 的 LSP、CSNP、PSNP   |
| **路由域认证** | ISIS 进程视图 | Level-2 的 LSP、CSNP、PSNP   |

### 4.2 配置示例

**接口认证：**

```
interface GigabitEthernet0/0/0
 isis authentication-mode md5 cipher Huawei@123
```



**区域认证：**

```
isis 1
 area-authentication-mode md5 cipher Huawei@123
```



**路由域认证：**

```
isis 1
 domain-authentication-mode md5 cipher Huawei@123
```



### 4.3 特殊参数 all-send-only

```
domain-authentication-mode md5 cipher Huawei@123 all-send-only
```



- 作用：**只发认证信息，不检查收到的报文**
- 别人发来的报文即使没有认证或密码错误，也照样接收
- 用途：**平滑升级**（先在不支持认证的设备上用 all-send-only 逐步开启）

### 4.4 认证机制说明

- 开启认证后，本端会对**收到的 LSP** 进行检查
- 没有携带认证信息或密码错误的 LSP → 忽略，不加入 LSDB
- 本端发送的 LSP 中会携带认证信息
- 没有开启认证的设备收到后，会正常加入 LSDB

## 五、沉默接口（Silent-Interface）

### 5.1 作用

将指定接口设置为沉默接口后，该接口：

- **不发送** ISIS 的 Hello 报文
- **不建立**任何 ISIS 邻居关系
- **但仍会**将该接口的直连网段通过 LSP 通告出去

### 5.2 使用场景

- 连接终端设备（如服务器、PC）的接口，不需要建立 ISIS 邻居
- 连接其他非 ISIS 网络设备的接口
- 避免不必要的 Hello 报文浪费带宽

### 5.3 基本命令

```
interface GigabitEthernet0/0/0
 isis silent
```

### 5.4 特殊参数 advertise-zero-cost

```
interface GigabitEthernet0/0/0
 isis silent advertise-zero-cost
```



**作用**：将该接口的直连网段以**开销值为 0** 的形式通告出去。

**不加 advertise-zero-cost 时**：以接口实际配置的 ISIS 开销值发布。

**使用场景**：

- 希望其他路由器优选这条路由（开销 0 表示最优）
- 特殊的流量引导需求

### 5.5 对比总结

| 配置                              | 发送 Hello | 建立邻居 | 通告网段 | 通告开销 |
| :-------------------------------- | :--------- | :------- | :------- | :------- |
| 普通接口                          | ✅          | ✅        | ✅        | 实际开销 |
| `isis silent`                     | ❌          | ❌        | ✅        | 实际开销 |
| `isis silent advertise-zero-cost` | ❌          | ❌        | ✅        | **0**    |

## 六、路由汇总

### 6.1 三种汇总场景

| 汇总位置             | 命令                                      | 作用                                   |
| :------------------- | :---------------------------------------- | :------------------------------------- |
| **Level-1 路由器**   | `summary 192.168.0.0 255.255.0.0 level-1` | 将本地明细汇总后，在 L1 区域内通告     |
| **Level-1-2 路由器** | `summary 192.168.0.0 255.255.0.0 level-2` | 将 L1 区域的路由汇总后，通告给 L2 骨干 |
| **Level-2 路由器**   | `summary 172.16.0.0 255.255.0.0 level-2`  | 将本地明细汇总后，在 L2 区域内通告     |

### 6.2 注意事项

- 汇总只在**始发路由器**上做
- 汇总后会产生一条指向 Null0 的**防环路由**
- 汇总路由的 cost 取所有被汇总明细路由中的**最小值**

### 6.3 举例

**Level-1 汇总：** R1 有三个 Loopback（192.168.1.254/24、192.168.2.254/24、192.168.3.254/24），在 R1 上汇总：

```
isis 1
 summary 192.168.0.0 255.255.0.0 level-1
```



**Level-1-2 汇总：** R4 作为 L1-2 路由器，将 R1 区域的 192.168.x.x 汇总后通告给 L2 骨干：

```
isis 1
 summary 192.168.0.0 255.255.0.0 level-2
```



**Level-2 汇总：** R6 有三个 Loopback（172.16.10.254/32、172.16.20.254/32、172.16.30.254/32），在 R6 上汇总：

```
isis 1
 summary 172.16.0.0 255.255.0.0 level-2
```

## 七、防环机制

### 7.1 核心规则

**Level-1 路由器不会把从 ATT=1 生成的默认路由再通告给邻居（包括 L1-2 路由器）。**

### 7.2 为什么需要这个机制？

如果不防环：

1. R1 生成默认路由指向 R2（L1-2）
2. R1 把这条默认路由写入 LSP 发给 R2
3. R2 从 R1 学到默认路由指向 R1
4. 当 R2 的出口失效时，流量会在 R1 和 R2 之间来回转发 → **环路**

### 7.3 实际行为

- R1 的默认路由**只在本地生效，不写入 LSP**
- R2 **永远不会从 R1 学到默认路由**
- 无环路风险

### 7.4 一句话总结

> 默认路由是**单向**的，只能从上往下（L1-2 → L1），不会反向传播。

## 八、快速总结

| 概念                | 一句话记住                                              |
| :------------------ | :------------------------------------------------------ |
| 骨干区域            | 连续 L2/L1-2 设备组成，区域号可以不同                   |
| 非骨干区域          | 区域号相同的 L1 设备组成                                |
| ATT 位              | L1-2 告诉 L1“往我走能出去”，L1 自动生成默认路由         |
| 路由渗透            | 把 Level-2 的明细路由主动引入 Level-1                   |
| 路由认证            | 接口认证（Hello）、区域认证（L1 LSP）、域认证（L2 LSP） |
| all-send-only       | 只发认证，不校验收到的报文，用于平滑升级                |
| 沉默接口            | 不建邻居，不通告 Hello，但通告网段                      |
| advertise-zero-cost | 沉默接口以开销 0 通告网段                               |
| 路由汇总            | 在始发路由器上做，减少路由条目，自动生成 Null0 防环     |
| 防环机制            | L1 的默认路由只本地用，不反向通告给 L1-2                |
